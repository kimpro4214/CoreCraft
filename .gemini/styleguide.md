# 코어크래프트(Core Craft) 자체 엔진 코드 리뷰 가이드라인

- **기본 설정**: 모든 PR 요약과 리뷰 코멘트는 '한국어'로 작성할 것.

- **모던 C++ 및 메모리 관리 (가장 중요)**:
  - 가비지 컬렉터가 없는 자체 엔진 환경이므로 **메모리 누수(Memory Leak)** 가능성에 가장 민감하게 반응해 줘.
  - RAII 패턴이 잘 지켜지고 있는지 확인해.
  - 소유권이 명확하지 않은 생 포인터(Raw Pointer) 사용을 지적하고, 적절한 스마트 포인터(`std::unique_ptr`, `std::shared_ptr`) 도입을 제안해 줘.
  - 잦은 메모리 할당/해제로 인한 오버헤드를 막기 위해 오브젝트 풀링(Object Pooling)이 필요한 곳을 짚어줘.

- **자료구조 및 알고리즘 최적화**:
  - 크래프팅 게임 특성상 수많은 아이템, 엔티티, 인벤토리 데이터를 다루게 됨. 이를 처리하는 자료구조(예: `std::vector`, `std::unordered_map` 등)가 최적의 형태로 사용되었는지 검토해 줘.
  - 불필요한 데이터 복사(값에 의한 전달 등)나 비효율적인 탐색(O(N) 순회 등)이 있다면 즉시 경고하고 최적화 방안을 제시해.

- **DirectX 11 렌더링 리소스 관리**:
  - COM 객체(예: `ID3D11Device`, `ID3D11Buffer`, `ID3D11ShaderResourceView` 등)의 생성과 해제(`Release()`, 혹은 `ComPtr` 사용 여부)가 안전하게 처리되었는지 점검해서 그래픽 리소스 누수를 막아줘.
  - 렌더링 루프 내에서 불필요한 상태 변경(State Change)이나 비효율적인 버퍼 업데이트가 발생하는지 확인해.

- **피드백 스타일**:
  - 단순히 "문법 오류가 있다"를 넘어, "이 구조는 메모리 단편화를 유발할 수 있으므로 이런 설계가 더 낫다"는 식의 깊이 있는 시스템 엔지니어링 관점의 피드백을 제공해 줘.
